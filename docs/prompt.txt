Este é o repositório Coup Master 3D.

Antes de mexer em qualquer código, leia obrigatoriamente:
- AGENTS.md
- README.md
- docs/GDD.md
- docs/TDD.md

Contexto geral:
O projeto é uma mesa 3D sandbox para jogar Coup Master no navegador, inspirada no Tabletop Simulator. Usa HTML, CSS, JavaScript, Three.js/WebGL, Rapier 3D e Firebase Authentication + Realtime Database. O foco atual é o MVP online casual, não ranqueado.

Arquivos principais:
- index.html
- css/three-board.css
- js/three/app.js
- js/three/boot.js
- js/three/config.js
- js/three/dom.js
- js/firebase/room-service.js
- js/firebase/auth-service.js
- js/firebase/login-page.js
- js/firebase/lobby-page.js

Regras importantes:
- Não misturar lógica Firebase dentro da renderização 3D.
- Manter app.js focado na mesa, interação, física, HUD e estado visual.
- Manter serviços online em js/firebase/.
- Não implementar loja, ranking, DLC paga, matchmaking ou ranqueado por enquanto.
- O modo casual é sandbox/manual, não deve automatizar todas as regras.
- Preserve alterações existentes e não reverta nada sem pedir.
- Sempre que alterar código JS, rode:
  node --check js\three\app.js
  node --check js\three\boot.js
  node --check js\firebase\room-service.js
  conforme os arquivos alterados.

Estado atual importante:
- index.html é a tela principal da mesa 3D.
- login.html tem login Google e visitante.
- lobby.html cria/entra em salas.
- Salas usam código curto de 4 caracteres.
- Jogadores ficam em rooms/{roomCode}/players/{uid}.
- Fechar/minimizar aba não libera o slot.
- Apenas o host/admin pode remover jogadores explicitamente.
- Há lista de jogadores no HUD, abaixo da barra superior direita.
- O modal de jogador mostra nome, slot, status e perfil; só host pode remover.
- Há tela de carregamento no index.html para evitar mostrar HUD/mesa vazia antes do boot terminar.
- O HUD tem barra superior esquerda com sair/reset, barra superior direita com utilidades, barra lateral esquerda com chat/histórico, status acima da barra inferior e barra inferior de ações rápidas.
- Chat em tempo real já existe.
- Modo espectador já existe e permite ver a mão de outro jogador se ele aceitar.
- Regras alternativas usam assets/img/guides/alternative-rules1.png até alternative-rules5.png.
- Sincronização multiplayer atual é por snapshots finais e eventos discretos, não por drag frame a frame.
